ARM架构 离线部署harbor镜像仓库踩坑

本文使用的是Github上拉取的harbor的离线部署包，压缩包内虽然带了install.sh，但是在实际的部署过程中的问题还是挺多的，特别记录下。

注意：1、本文主要安装v1.9.3版本的harbor；2、本次记录的也是单机版的harbor，有兴趣的可以搞一个双机热备或者HA，步骤和x86差不多了

环境部署包准备

  • 部署包：harbor-offline-installer-v1.9.3-aarch64-kylin.tar.gz

    主要目录如下：

  • docker 环境
  • docker-compose 已存在

其中harbor所需要的镜像都在harbor.v1.9.3.tar.gz这个文件里了

部署试运行

这一步的目的主要是因为，如果把部署包直接放到服务器上，install.sh安装的话，部分容器会无法正常运行。会提示容器运行失败，授权过期的问题。

harbor需要的主要镜像如下：

goharbor/harbor-log:v1.9.3

goharbor/registry-photon:v2.7.1-patch-2819-2553-v1.9.3

goharbor/harbor-registryctl:v1.9.3

goharbor/harbor-db:v1.9.3

goharbor/harbor-core:v1.9.3

goharbor/harbor-portal:v1.9.3

goharbor/harbor-jobservice:v1.9.3

goharbor/redis-photon:v1.9.3

goharbor/nginx-photon:v1.9.3

goharbor/clair-photon:v2.1.0-v1.9.3

docker启动harbor

docker-compose down && docker-compose up -d重启即可。

建议直接做成service，用systemd来管理harbor启动，这样就不用每次进入目录通过docker-compose来控制启停了。
